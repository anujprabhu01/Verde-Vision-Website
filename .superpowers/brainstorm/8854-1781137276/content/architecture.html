The Island Model — what actually loads JS

client:load

Loads immediately. Used for above-the-fold interactive elements.

  • BASlider (hero)
  • BookingWidget
  • Nav toggle

client:visible

Loads only when scrolled into view. JS for off-screen sections never runs until needed.

  • ScrollPath (Why It Wins)
  • VideoControls
  • CTATree
  • Canvas grid

What this means in practice

Right now, all 42KB of JS loads and runs immediately on page load — including the CTA tree animation, canvas grid, and scroll path that the user hasn't scrolled to yet. With client:visible, those islands don't even download until the user reaches them. On a slow mobile connection, the initial page load becomes dramatically faster.
