Migration Strategy

How we get from the current site to Astro without breaking the live site at any point.

Branch strategy — current site stays live throughout

main Current site — stays live on useverdevision.com throughout
astro-migration New branch — Vercel gives this a preview URL automatically
merge → main Only when fully tested and approved — then useverdevision.com updates

Order of operations

1
Scaffold + configure — npm create astro, install fontsource packages, configure Vercel adapter, set up PostCSS/Autoprefixer
2
Layout.astro — base HTML shell, <head>, meta tags, OG tags, fontsource imports, global CSS tokens
3
Nav + Footer — static components first, no interactivity needed yet. Establishes the shell of the page.
4
Hero — static markup first (headline, CTAs), then add BASlider as a client:load island with Pointer Events API replacing the old mouse/touch handlers
5
Why It Wins — migrate the scroll path animation as a client:visible island. Refactor scroll listeners → cached rAF loop
6
Features — swap <img> tags for Astro <Image>. Move all 5 feature PNGs to src/assets/. This is where the biggest single performance win happens.
7
In Action — video section + controls as a client:visible island. Video itself stays on CDN (r2.dev), only the JS controls are an island.
8
Booking — calendar + form as a client:load island. Logic is self-contained, moves cleanly.
9
CTA + canvas grid — CTA tree as client:visible, canvas grid as client:idle. Last to load, least critical.
✓
Cross-device audit — test on iOS Safari, Android Chrome, desktop Chrome/Firefox/Safari. Fix any remaining inconsistencies.
✓
Merge to main — Vercel auto-deploys. useverdevision.com updates. Done.

What stays exactly the same

  • All copy and content
  • Visual design direction (colors, typography, layout)
  • Domain (useverdevision.com)
  • GitHub repo (anujprabhu01/Verde-Vision-Website)
  • Vercel auto-deploy on push to main
  • All existing assets (logo, headset image, video CDN URL)
