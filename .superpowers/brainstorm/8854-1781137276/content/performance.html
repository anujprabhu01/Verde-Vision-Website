Performance Strategy

The three biggest fixes — images, fonts, and CSS. Each is automatic after the migration.

1. Images — the #1 performance killer

Before (current)

  • feature-01-design.png — 2.8MB
  • feature-02-blueprint.png — 2.7MB
  • feature-03-estimate.png — 2.6MB
  • feature-04-persist.png — 2.5MB
  • feature-05-crew.png — 3.6MB
  • HeadsetTransparent.png — 2.1MB
  • Before.png / After.png — 4MB
Total: ~20MB loaded on every page visit

After (Astro Image)

  • feature-01-design.webp — ~50KB
  • feature-02-blueprint.webp — ~50KB
  • feature-03-estimate.webp — ~50KB
  • feature-04-persist.webp — ~50KB
  • feature-05-crew.webp — ~50KB
  • HeadsetTransparent.png — ~200KB (PNG preserved, transparency)
  • Before/After — ~150KB
Total: ~600KB — 97% reduction, zero manual work

Astro's <Image> component handles conversion, resizing, lazy loading, and correct width/height attributes (eliminates layout shift) automatically.

2. Fonts — eliminate 3 external network requests

Before

3 Google Fonts stylesheet requests → browser blocks render until fonts load → flash of unstyled text on slow connections. Each request is a DNS lookup + TLS handshake + download.

After

Use @fontsource npm packages — fonts bundle into your Vercel deployment. Zero external requests. No FOUT. Works offline. No Google tracking. Same fonts, served from your own CDN.

npm install @fontsource/inter @fontsource/fraunces @fontsource-variable/geist-mono

3. CSS — from 70KB global to scoped-per-component

Before

  • Single 70KB stylesheet loaded on every page
  • Styles bleed between sections
  • No way to know what's safe to delete
  • Specificity wars from iterative tweaks
  • Cross-device inconsistencies from missing vendor prefixes

After (Astro scoped CSS)

  • Each .astro file has its own <style> block — scoped automatically
  • CSS variables/tokens preserved in global.css (~2KB)
  • Unused CSS per component = impossible (scoped = only what you write)
  • PostCSS + Autoprefixer runs automatically via Vite
  • Dead code elimination built in
