Which tech stack should we migrate to?

All three solve the performance and cross-device issues — they differ in scope and approach. Pick what feels right.

A

Vite + TypeScript

Add a proper build pipeline to the existing vanilla HTML/CSS/JS. Keeps the same mental model.

Wins

  • Smallest scope change
  • Image optimization (vite-imagetools)
  • CSS purging, autoprefixing, minification
  • JS tree-shaking & bundling
  • TypeScript for fewer runtime bugs

Limits

  • No component system — CSS still global
  • Cross-device JS bugs need manual fixing
  • 70KB CSS still needs manual cleanup
B

Astro ⭐ Recommended

Purpose-built for marketing/content sites. Ships zero JS by default. Interactive parts (slider, calendar) become isolated "islands."

Wins

  • Built-in image optimization → 3.5MB PNGs become ~50KB WebP automatically
  • Zero JS shipped unless needed (huge perf gain)
  • Scoped CSS per component → no more global bleed
  • Google Fonts inlined → no layout shift
  • Native Vercel adapter (one-line deploy)
  • Can use React/Svelte for interactive islands

Limits

  • New file format to learn (.astro)
  • Full rewrite (1–2 days of work)
C

Next.js App Router

React-based, deep Vercel integration. Industry standard for production web apps.

Wins

  • next/image → automatic optimization
  • next/font → zero layout shift fonts
  • Huge ecosystem (any library available)
  • Perfect Vercel integration
  • React component system

Limits

  • Overkill for a single marketing page
  • React runtime adds ~40KB to every page
  • App Router has its own learning curve
  • More complexity than this site needs

Why Astro is the right call here

The #1 performance killer on this site is the unoptimized images (~15MB total). Astro fixes that automatically. The cross-device bugs come from hand-rolled JS interactions — Astro's island model means you only ship JS for components that need it, and you can use battle-tested React libraries for the interactive parts. Vite is the build engine under the hood anyway, so you get all of Option A's wins plus a component model that fixes the CSS bleeding issues.
